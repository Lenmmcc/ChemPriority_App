# 一键批量查询结果总览设计

## 目标

将“一键批量查询”页面现有的单一结果表选择器和用途图表选择器，改为按查询模块组织的结果总览，使用户能在同一页面查看已选项目的关键结果、数据预览与可用图表。

## 范围

页面按结果是否存在动态显示以下模块：

1. 本地筛查：化学类型、DBE、VK、DF 和峰面积相关结果。
2. 标识符补全：补全结果与提示记录。
3. EPI Suite：主结果、原始结果和错误记录。
4. EPA CompTox：汇总、候选用途、错误记录及现有 EPA 用途图。
5. ECHA：REACH 用途、GHS/C&L 危害及对应明细/错误记录；存在 ECHA 用途图时同时显示。
6. 来源属性：汇总、证据和错误记录。
7. Pov-LRTP / PBM / ToxPi：仅在用户勾选并产生结果时显示。

未运行或没有可展示结果的模块不显示空白标签。结果仍保留在现有 `AutoWorkflowResult.tables` 中，ZIP 内的工作簿与图表文件结构不变。

## 交互设计

在“运行日志”之后增加“结果总览”标签区。每个标签按固定顺序显示：

- 一句结果说明或关键计数；
- 与该模块相关的表格预览；
- 该模块已有的图表，以模块内选择框切换；
- 错误和审计表折叠展示，避免占用主视图。

保留原始“查看结果表”选择器，作为所有原始工作表的完整浏览入口；原有 ZIP 下载按钮不变。

## 实现边界

- 不改变任何查询、计算、缓存、数据表或导出协议。
- 不新增外部网络请求；预览仅复用本次工作流已生成的表和图。
- 不新增本地筛查、EPI、GHS 或来源属性的图形算法；这些模块先以关键表格预览集成。现有 EPA/ECHA 用途图按所属模块呈现。
- 页面展示逻辑提取为纯辅助函数，以便通过文本/结构测试验证标签分组和图表归属。

## 验证

新增页面契约测试，断言结果总览覆盖各模块标签、保留原始表格入口和 ZIP 下载，并将 EPA/ECHA 图表按模块显示。运行 `python -m unittest discover -s tests -v` 与 `python -m compileall app.py pages src`。
