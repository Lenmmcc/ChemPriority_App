# ChemPriority 新用户文档设计

## 目标

重构项目根目录的 `README.md`，并新增 `docs/用户操作手册.md`，使首次接触 ChemPriority 的用户能够在不阅读源代码的前提下安装、启动、准备数据、运行四个业务模块、下载结果并理解主要限制。

## 文档边界

- 面向普通使用者的内容使用中文，采用可直接复制的命令、字段名和下载文件名。
- `README.md` 是仓库首页入口：说明项目用途、最短启动路径、四个模块速查、输入字段速查和详细手册入口。
- `docs/用户操作手册.md` 是唯一的详细操作说明：按“准备数据—执行操作—获取结果—处理异常”的顺序覆盖每个页面。
- 管理员配置与普通用户操作分开。仅说明 `CHEMSPIDER_API_KEY` 的 Streamlit Secrets 键名和安全原则，不写入真实密钥。
- 不修改应用代码、测试、现有未跟踪的测试说明或结果文件。

## README 信息架构

1. 项目定位、适用场景与四个相互独立的模块。
2. 五分钟本地启动：创建或启用环境、安装 `requirements.txt`、运行 `streamlit run app.py`。
3. 从首页到各页面的推荐使用路径，以及每个模块的输入、主要结果和适用条件。
4. Excel 通用规则与最小字段表；链接到 `data/data.xlsx` 和详细用户手册。
5. 管理员可选配置：Streamlit Secrets 中的 ChemSpider 密钥，明确其只影响标识符补全且缺失时不会阻断其他功能。
6. 结果使用边界、外部数据源依赖和人工复核要求。

## 用户操作手册信息架构

1. 使用前准备：浏览器、网络、Excel 文件结构、通用字段和会话数据保留规则。
2. 推荐工作流：已有毒性数据时直接进入 ToxPi；只有结构或标识符时按“标识符补全—用途查询”进行；需要环境归趋或 ADMETlab 输入整理时使用对应页面。
3. 四个模块的分步操作：
   - ADMETlab 输入校验和已校验输入表下载；说明当前页面不自动获取 ADMETlab 预测结果。
   - ToxPi 指标选择、权重、分组、随机种子和结果下载；解释高分只表示当前指标与权重下的相对优先级。
   - EPI Suite 自动预测、输入包备用路线、外部结果回传解析及失败记录。
   - 化合物用途查询的标识符补全、EPA CompTox Dashboard、ECHA CHEM、用途风玫瑰图和三份工作簿下载；区分来源、候选用途与证据。
4. 结果文件清单：每个文件的来源、主要工作表或内容、使用建议。
5. 故障排查：缺列、空值、外部服务超时、无法匹配标识符、查询部分失败、页面切换与清空数据。
6. 数据来源、隐私与解释限制：外部来源可能变化；输出用于筛选和整理，不替代人工证据核验或正式风险结论。

## 正确性与可维护性要求

- 所有模块名称、字段、控制项、默认外部来源和下载文件名必须与 `app.py`、`pages/` 和 `src/` 当前实现一致。
- 不将外部服务描述为保证可用，不将候选用途、预测值或 ToxPi 分数描述为最终监管结论。
- 两份文档避免大段重复：README 保留概览和链接，详细步骤只写入用户操作手册。
- 文档完成后检查 Markdown 标题层级、内部相对链接、命令、字段名和下载文件名；运行 `git diff --check` 检查格式错误。
